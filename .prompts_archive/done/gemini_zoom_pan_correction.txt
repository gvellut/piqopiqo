Previous update, I requested this (among other things) :

- panning means holding the left mouse button and moving the mouse. Doing that will change the part of the image that will be displayed on screen, similar to a panning on a map interface. If moving the mouse to the left, part of the image on the right side will get in view. 
- there can be no panning on the base state of the fullscreen mode, only when zoomed in.
- There is empty space added around the image. It is a value in pixels (in the space of the screen). This can be set in the config. So you can pan beyond the image. Because there can be black bars at the beginning (to maintain the ratio), that empty space can be be bigger for that dimension (it is maintained when zooming in but not added to the space set in the config ; if the config space is bigger, it takes precedence).

(look at the file called prompts/gemini_zoom_fullscreen.txt for the full request)

On the left side, the left is a bit bigger than bottom + top (both identical). On the right side, it is smaller than on the left side and also smaller than bottom + top.

The only discontinuity allowed is when getting back to the base state (see prompts/gemini_zoom_fullscreen.txt for what I mean). Also, if the base state has black bars (to maintain ratio) on a side, that is allowed to have bigger empty space on the way of zooming in : it stays the same in screen space but "diminishes" in image space when zooming in. however, if the space configured becomes bigger than that space of the base state, it should become the empty space.
