These modules are used for: 
- initializing a folder on open : they extract the EXIF data into a metadatad DB (so updating is fast ; updating done in src/piqopiqo/metadata/metadata_db.py), they also generate cached thumbnails (extraction existing from EXIF OR computing scaled down from the full image)
- they can be used to regenerate a thumbnail, or handling Refresh a folder in case of addition or deletion of some files (currently done explcitly by the user though commands in the GUI)
- the exif.py is used : to read the EXIF data on demand (when images are selected), to write an exif (when "Save exif" command is selected by the user)

The EXIF reading is done by pyexiftool, a Python helper on top of the exiftool command-line tool (which has to be installed before hand). It runs exiftool in batch mode (ie starts the exiftool process, then communicates with it using local IO). Currently, EXIF is extracted one image at a time (but could work with multiple).

I want to arrange it in a few ways :

    - there is no need for a lock for the pyexiftool process (no thread, only mutiple processes)

Some image itmes must be on top of priorities, some must go down.
