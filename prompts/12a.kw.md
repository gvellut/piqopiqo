Add a keyword tree : to select the keywords in a tree
- new Qt component in src/piqopiqo/panels/
- The keyword field is a string : DBFields.KEYWORDS in src/piqopiqo/exif_loader.py
- Add a button below the Keyword field in the src/piqopiqo/panels/edit_panel.py panel => "Open Keyword Tree"
- The tree is a hierarchical list of predefined keywords
- Open a modal Dialog with a OK Cancel buttons : if OK => the keywords metadata field is updated ; If cancel it is not. 
- The Keyword tree is a normal tree : with root : Keywords (added by the application : not really a keyword : it does not have a checkbox) :  Below are the list of keywords. Each item has a checkbox and is collapsible. Checking a keyword does not check its children (they are independent)
- the keyword checkboxes are initialized from the keyword fields of the edit panel. If a keyword of the tree is present in the field : it should be checked. If not, it should be unchecked. If multiple grid items were selected : the Metadata Keyworld fields may have been different and marked as <Multiple values> BUT in the keyword tree : the fields that are in some of the items but not all : they are marked in gray checkboxes. 
    - Modification : CLicking the gray checkboxes will add the keywords to all (and make the checkbox Black). Clicking a second time will remove the keywords from all (and make the checkbox blank). Clicking on a checkbox that was blank will add the keyword to all the grid items. Clicking on a keyword that was black : will remove the keyword from all the items
    - If no grid item had the keyword (checkbox originally blank) : clicking the checkbox will make the checkbox black and the keyword will be added to all.
    - If all the grid items had the keyword  (checkbox originally black) : clicking the checkbox will empty the checkbox and the keyword will be removed from all.
    - When saving (with OK) : if multiple grid items were selected when opening the tree : the keywords that were not modified will be left alone : if they were there for a grid item, they will stay there ; if they were not there, they will not be added. Only the modified keywords willd be added or deleted using the process described above.
- Additionnaly : there is a "Additional keywords" on the same level as the Keywords root (not really a root if the Additional keyuwords is there) in the tree (at the end ; not before the predefined keywords) : it will show the keywords that are in the fields (one or more) but are not predefined in the tree. It does not have a checkbox. It is not there if there are no additional keywords. The keywords below have a checkbox and the checkboxes follow the same rule as the predefined keywords in the tree

- At the top the keyword widget there is a button : "Import"  that will import a txt file (in Adobe Bridge format). With different levels indicated with different number of Tabs.
The No-tab keywords are directly below the Keyword tree item mentioned above.
The imported file is merged to the current predefined keyword tree (if empty : it is inialized) below the "Keywords" item

The tree is alphabetically ordered at each level (the siblings are ordered) so no possiblity to reorder manually by dragging and dropping.
The tree (list of keywords) is saved to the Application support directory inside a JSON file called keyword-tree.json so usable between sessions of the application. It is saved when importing and when new items are added or items are deleted (see below). The Additional keywords section is never saved to that file : it is always created at runtime depending on the keywords field of the selected items.
Add an option in src/piqopiqo/config.py so that the tree is not loaded and saved : DETACHED_KEYWORD_TREE => neither laoded nor saved : just of the current session (for testing)

- It is possible to manipulate the tree interactively : right click on a item (including the Keywords item) : there are 2 choices : Add keyword; Delete keyword (the latter is not present for the Keywords item ; none of them are present for the Additional keywords and its children) 
- When delete keyword is chosen : there is a confirmation dialog. Also if the keywords are present in the selected grid item keywords (so checkmarks displayed in black or gray) => they are added to Additional keywords with the same type pofcheckmark (but NOT if not there : they simply vanish)
- When Add keyword is selected : a new item is created just below the item on which Add keyword was clicked : its label is editable : entering something and Enter will finish creating it : the item is moved to its position in alphabetical order below the item on which Add keyword was clicked. The item stays visible when moved (so need to scroll the view to keep it visible). If Esc is pressed while editing : the new keyword field disappears and the keyword is not created. If the user types " in the keyword : it is auto-removed when Enter (no dialog or warning).

- keywords can have a comma , inside : it should not confused as a separator for the keyword items as displayed in the edit panel. So if there is a , in a keyword value : it should be surrounded with quotes ". Validate that the quotes are balanced before saving. The field can end with a ,. It is valid to have multiple ,. So make sure to change the reading from the exif in src/piqopiqo/exif_loader.py to allow for it.
    - it is assumed there can be no " in the keywords (so no need for escape)
    - When read from the EXIF : auto-remove the " from each value (if list) (no dialog or warning).
