In fullscreen mode, add support for zoom and unzoom through mouse wheel, as well as support for panning using pressed left mouse button and moving the mouse :
- the base state is what is currently displayed when entering fullscreen mode : zoom out cannot go beyond that, only zoom in. Then when zoomed in, the zoom out can be used to get back
- when mouse wheel up, zoom in : add a zoom factor in the Config : this will be the factor to zoom : by default, it will be 1.5 . So if in the base state, a physical pixel of the screen, was occupied with 3x3 pixels of the image, after the zoom in operation, the physical pixel will be occupied by 2x2 pixels of the image. Or maybe, if my reasoning is incorrect, do the zoom in somewaht in that direction, make a note of it, and I wil change later. However there is one special state : when 1 pixel of the screen is occupied by 1x1 pixel of the image (ie "real" size). In the process of zooming in, if that state would be passed, the zoom in  must not be the full factor and instead that special state must be set after that zoom in (one turn of the wheel). Also log debug that state. For example , if 1 pixel of the screen is occupied by 1.2x1.2 pixels of the image, the next zoom in must be the special state.
- The max zoom in should be in the config : for now set it to 3 ie 1 pixel of the image occupies 3x3 pixels of the screen.
- when mouse wheel down, : zoom out. In that direction, the "real" size is also a special state. Also the base zoom of the image (when entering full screen) is also a special state. So the zoom out cannot go smaller than that special state.
- In all cases, always maintain the ratio : no deformation / skew of the displayed image.
- when zoomed in, if using the keyboard presses left or right arrow, the zoom does not transfer to the next image : it is displayed in the base state for fullscreen mode (the one that exists now)
- if getting out of fullscreen, if entering on the same photo, the zoom state of the photo is not preserved : it always starts with the base state.

- The position of the mouse pointer is also important : when zooming in or out using the mouse wheel, the pixel of the image below the mouse must stay at the same position on the screen. So the zoom is centered on that point. The exception is when zooming out all the way to the base state, the image must go back to being centered like it is now when entering the fullscreen mode. Another exception is if the mouse is not above a pixel of the image (for example, at the beginning, there may be black bars ie background color around the image if too small or if the ratio must be maintained or the mouse may be on another screen, even though the focus is still on te fullscreen widget), you must clamp the mouse pointer position as if it were on the closest position above the image (the image may be bigger than the screen so that position might be in coordinates outside the screen).

- panning means holding the left mouse button and moving the mouse. Doing that will change the part of the image that will be displayed on screen, similar to a panning on a map interface. If moving the mouse to the left, part of the image on the right side will get in view. 
- there can be no panning on the base state of the fullscreen mode, only when zoomed in.
- There is empty space added around the image. It is a value in pixels (in the space of the screen). This can be set in the config. So you can pan beyond the image. Because there can be black bars at the beginning (to maintain the ratio), that enpty space can be be bigger for that dimension (it is maintained when zooming in but not added to the space set in the config ; if the config space is bigger, it takes precedence).

- Add a mouse wheel sensitivity factor for zoom in the config : for example 2 meaning you must roll twice to count as a zoom in or out event for the software. If you only roll once (within a window of time: not usre if handled by Qt or must be handled entirely by the program), no zoom in or out opereation must be performed.
