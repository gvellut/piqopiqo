The Best Solution: Use QKeySequence

Instead of parsing strings and mapping them to Enums manually, let Qt do the work. QKeySequence can convert a string (like "Ctrl+=") into a key code, and you can convert the QKeyEvent into a key code to compare them.

Here is how you should rewrite _match_shortcut:
code Python

from PySide6.QtGui import QKeySequence, QKeyEvent
from PySide6.QtCore import Qt

def _match_shortcut(event: QKeyEvent, shortcut_str: str) -> bool:
    """
    Check if a key event matches a shortcut string using QKeySequence.
    Handles 'Ctrl+=', 'Shift+A', etc. automatically.
    """
    # 1. Parse the string into a Qt KeySequence (e.g. "Ctrl+=")
    expected_sequence = QKeySequence(shortcut_str)
    
    # 2. Convert the current event (Key + Modifiers) into a generic Qt Key combination
    #    Note: We rely on event.key(), not event.text()
    event_key_combination = event.key() | event.modifiers()
    
    # 3. Compare
    return expected_sequence == QKeySequence(event_key_combination)
