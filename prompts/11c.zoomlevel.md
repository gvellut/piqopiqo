    def set_1_to_1_pixel_mode(self):
        """
        Sets the zoom level so 1 Image Pixel == 1 High-DPI Buffer Pixel.
        """
        # 1. Get the screen this window is on
        current_screen = self.screen()
        if not current_screen:
            return

        # 2. Get the Device Pixel Ratio (e.g., 2.0 on Retina)
        dpr = current_screen.devicePixelRatio()
        
        # 3. Calculate the scale factor
        # If DPR is 2.0, one logical point is 2 pixels.
        # To get 1 image pixel = 1 physical pixel, we must scale by 0.5.
        scale_factor = 1.0 / dpr
        
        print(f"Setting 1:1 Hardware Zoom.")
        print(f"Screen DPR: {dpr}")
        print(f"Applied Scale: {scale_factor}")

        # 4. Apply scale
        self.resetTransform()
        self.scale(scale_factor, scale_factor)


    Why this works (The Math)
    Qt Coordinates: On your Mac, Qt thinks the window is, for example, 2560 units wide.
    OS Coordinates: The macOS WindowServer has allocated a texture 5120 pixels wide for that window.
    Standard Draw: If you draw a 100px image at scale 1.0, Qt draws it across 100 logical units, which fills 200 pixels in the texture (interpolation happens).
    The Fix: By setting self.scale(0.5, 0.5), you tell Qt to draw the 100px image across 50 logical units.
    Result: 50 logical units * 2.0 DPR = 100 pixels in the texture.
        Image: 100px
        Texture: 100px
        Mapping: 1:1 (Perfect sharpness).
