
- create photo list model : Not a simple tuple. to which filters / sorts / selection are applied : and taken as backing data for the rest of the application
- So take into account the folders, the path, the exif, the thumbnail
- This model is manipulated : sorted +  added + photos selected : this is what is displayed and the underlying data manipulated by the application
- handle deletion from the model : metadata must be removed from db + thumbnail removed from cache.
- handle addition : metadata must be set from exif + thumbnail generated.

- Add a menu : View : with section Sort : and submenu Time taken, File name, File name by folder => difference between the last 2 is that the last one takes into account the folder name, the other only the base filename
- Add to menu View a section : Refresh. This is so if a new file was added or if a file was removed by the user outside the application : its metadata + thumbnail must be processed (added or removed)

- Add to photo cell : a right click menu with entries : 
    - Duplicate : creates a new photo file. It adds a " copy" suffix to the filename (before the extension) in the same folder (if " copy" exists, " copy2" must be tried and so on 2 3 ... until one is found). The photo is processed as an addition.
    - Move to trash : the photo is moved to the macos trash. It is considered deleted from POV of application.
- On right click on a photo : the photo is selected if it was not.
- If multiple photos are selected and the click is on one of the selected photos : the command applies to all the photos. If the click is not on one of the selected photos : the photos are deselected and the rightr-click photo is selected. The command will only apply to the newly selected photo.
- the model is updated : the new photo is inserted, or the photo is deleted => it is displayed or removed from the grid. The sorting order must be maintained. If Duplicate : keep the clicked photo visible. If Move to trash : keep the first non-deleted photo after the clicked photo visible. If there is none (all the photos until that one to the last are deleted) : make the last photo visible
- the process inside thumb_man.py and exif_man must run to add what is needed for the new photo. If deleted, the entry for the photo must be deleted from the metadata db + thunmbnail removed from cache.
